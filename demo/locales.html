设计理念

Gemini UI 根据 Google 账号语言显示不同文本。原代码硬编码繁体中文+英文，简体中文用户直接报错。新方案将所有语言文本集中到 skills/lib/locales/gemini.js，核心逻辑不再硬编码任何语言字符串。

自动检测 + 手动覆盖

🔍 自动检测

优先级：navigator.language → <html lang> → 按钮文本反向推断 → null (回退 fuzzy)

🎛️ 手动覆盖

$ node index.js --locale=zh_CN "prompt"

未知 locale 自动使用 FUZZY 正则匹配全部已知文本

验证

$ node -e "const L=require('./skills/lib/locales/gemini'); L.setLocale('zh_CN');
console.log(L.verifyText('modelVerify'));"  → "Pro扩展"

$ node -e "L.setLocale('en'); console.log(L.verifyText('modelVerify'));"  → "Pro Extended"

$ node -e "L.setLocale(null); console.log(L.txt('extended'));"  → /扩展|延長|Extended|拡張/i

技术细节：page.evaluate 中 RegExp 传递

L 模块是 Node.js 作用域，返回的 RegExp 对象不能通过结构化克隆传到浏览器端（Playwright 只支持 JSON 可序列化类型）。解决方案：在 Node.js 侧提取 source + flags 字符串，浏览器侧 new RegExp(src, flags) 重建。

const _extRe = L.txt('extended');
const idx = await page.evaluate(({extSrc, extFlags}) => {
    const re = new RegExp(extSrc, extFlags);
    ...匹配DOM...
}, {extSrc: _extRe.source, extFlags: _extRe.flags});
